Quick Start

' Minimal hello-world program.
#include once "vt/vt.bi"

vt_title "Hello"
vt_screen VT_SCREEN_0       ' 80x25, 8x16 font

vt_color VT_YELLOW, VT_BLACK
vt_print_center 12, "Hello, World!"

vt_color VT_LIGHT_GREY, VT_BLACK
vt_print_center 14, "Press any key..."

vt_sleep 0   ' wait for keypress

vt_screen

Open or reopen the virtual text screen. Calling it again closes and reinitializes with the new settings.

Function vt_screen(mode As Long = VT_SCREEN_0, flags As Long = VT_WINDOWED, pages As Long = 1) As Long
ParameterTypeDefaultDescription
modeLongVT_SCREEN_0Screen mode. One of the VT_SCREEN_* constants.
flagsLongVT_WINDOWEDWindow behaviour flags. One or more VT_WINDOWED / VT_FULLSCREEN_* constants combined with Or.
pagesLong1Number of cell buffers to allocate (1–8). Page 0 is always VT_VIDEO. Extra pages are used with vt_page / vt_pcopy.

Return value

0 = success
-1 = SDL_Init failed
-2 = SDL window creation failed
-3 = SDL renderer creation failed
-4 = font texture creation failed
-5 = page buffer memory allocation failed

Call vt_title before vt_screen to set the window title before the window appears.
Call vt_scrollback after vt_screen to enable the scrollback buffer.
Call vt_copypaste after vt_screen to enable selection/clipboard support.

vt_on_close

Register a callback function that is invoked when the user clicks the window [X] button. Without a callback the default behaviour is to shut down immediately and call End — the program exits with no chance to intervene. With a callback registered the library calls your function first and acts on the return value.

Sub vt_on_close(cb As Function() As Byte)
ParameterTypeDescription
cbFunction() As Byte Pointer to a Function() As Byte. Pass 0 to remove the callback and restore the default auto-close behaviour.

Callback return value

0 — allow the close: library calls vt_shutdown() then End, identical to the default.
1 — veto the close: window stays open, the event is discarded, program continues normally.
VT functions are safe inside the callback. The SDL event-pump reentrancy guard is released before your callback is invoked and re-acquired afterwards, so vt_inkey, vt_getkey, vt_sleep, vt_present, and all drawing functions work correctly. This means you can draw a confirmation dialog and wait for a keypress directly inside the callback.

If your callback calls vt_shutdown() and End explicitly, the re-acquire never runs — that is fine. 
' Declare the callback with the exact signature:
Function on_close() As Byte
    ' Ask the user before closing:
    vt_color VT_BLACK, VT_LIGHT_GREY
    vt_locate 25, 1
    vt_print "  Unsaved changes!  Quit? (Y/N)" & String(48, " ")
    
    Dim ans As String = vt_getchar("YyNn")
    If LCase(ans) = "y" Then Return 0   ' allow close

    ' Restore UI and veto:
    draw_statusbar()
    vt_present()
    Return 1
End Function

' Register after vt_screen:
vt_screen VT_SCREEN_0
vt_on_close(@on_close)

' Remove callback (restore default auto-close):
vt_on_close(0)

ANSI escape sequence opt-in opt-in

When #Define VT_USE_ANSI is placed before #include once "vt/vt.bi", vt_print parses ANSI / VT100 escape sequences before writing to the cell buffer. Unknown final characters are emitted as raw CP437 glyphs rather than silently dropped, making debugging easier. Without VT_USE_ANSI, vt_print does a plain byte feed with CRLF collapse only.

Supported sequences:

SequenceNameAction
ESC [ <params> mSGRSet colour and blink attributes. Params follow standard ANSI SGR numbering.
ESC [ row ; col H
ESC [ row ; col f		CUPPosition cursor (1-based). Omitted values default to 1.
ESC [ 2 JEDClear screen. Only param 2 is supported in this release.
Use Chr(27) or !"\x1b" to embed escape in a string literal. Do not use !"\e" — FreeBASIC silently drops \e in bang-strings. 
#Define VT_USE_ANSI
#include once "vt/vt.bi"

' SGR: red on black, then reset
vt_print !"\x1b[31mHello\x1b[0m World"

' CUP: move to row 5, col 10, then print
vt_print !"\x1b[5;10HPositioned here"

Scroll Region & Scrollback

  • vt_scroll — move scrollback view
  • vt_screen_to_live_row — map screen row during scrollback

vt_scroll

Move the scrollback view by a number of lines. Positive values scroll toward older history; negative values scroll toward the live view. Scrollback must be enabled with vt_scrollback first. Does not call vt_present.

Function vt_scroll(amount As Long) As Byte
ParameterTypeDefaultDescription
amountLongnoneLines to scroll. Positive = back in history, negative = toward live view. Clamped to valid range automatically.

Return value

1 = offset changed
0 = already at limit, amount is zero, or scrollback is disabled
vt_scroll is called automatically by the library when the user presses the appropriate scrollback keys. You only need to call it directly if you want to drive scrollback from your own input (e.g. a custom mouse wheel handler).

see Internal Keyboard Bindings for the full table.

vt_screen_to_live_row

Translate a screen row number to the corresponding live buffer row while the scrollback view is offset. Useful when implementing click-to-locate functionality: call this before snapping back to the live view with vt_scroll(-99999).

Function vt_screen_to_live_row(screen_row As Long) As Long
ParameterTypeDefaultDescription
screen_rowLongnone1-based screen row as seen during the current scrollback offset.

Return value

1-based live buffer row that corresponds to screen_row.
0 if the screen row is showing pure scrollback history with no live equivalent.
Pass-through (returns screen_row) when scrollback is not active or the row is a fixed row outside the scroll region. 
' Click-to-locate pattern during scrollback:
Dim lr As Long = vt_screen_to_live_row(clicked_row)
If lr > 0 Then
    vt_scroll -99999   ' snap back to live
    vt_locate lr, clicked_col
End If

vt_view_print

Restrict the scroll region and print region to a range of rows and/or columns (like QBasic's VIEW PRINT, extended with column bounds). Rows and columns outside the active viewport are unaffected by vt_cls, vt_print, and scrolling. Call with no arguments (or all -1) to reset the full viewport including column bounds.

Sub vt_view_print(top_row As Long = -1, bot_row As Long = -1, lft_col As Long = -1, rgt_col As Long = -1)
ParameterTypeDefaultDescription
top_rowLong-1First row of the scroll region, 1-based. -1 resets all four bounds to the full screen.
bot_rowLong-1Last row of the scroll region, 1-based. Must be ≥ top_row.
lft_colLong-1First column of the print region, 1-based. -1 (default) means full screen width. When active, vt_print newlines land at this column.
rgt_colLong-1Last column of the print region, 1-based. Must be ≥ lft_col. When active, vt_print wraps at this column.
vt_locate is always absolute — column bounds do not affect it.
Calling with all -1 resets the entire viewport: row bounds, column bounds, and cursor position.

vt_sort

Sort a 1D array in-place using Shellsort. Overloaded for all numeric types (Byte, UByte, Short, UShort, Long, ULong, LongInt, ULongInt, Single, Double) and String. The compiler selects the correct overload automatically — a single call works for every supported type. Arrays with any LBound are handled correctly.

Two call forms are available: a direction constant, or a comparator callback for custom ordering.

ZString not supported. ZString * N cannot be covered by a generic overload because the fixed length N is part of the type in FreeBASIC — every distinct size would require its own overload, which is not feasible. ZString is a C-interop type; data intended for sorting should be stored as String. If you do have a ZString * N array to sort, use the index-sort pattern: sort a Long index array with a comparator that reads the zstring array via the index value, then read results in index order — no physical rearrangement of the zstring array is needed.

Form 1 — direction constant

ParameterTypeDescription
arr()any supported 1D arrayArray to sort in-place.

Form 2 — comparator callback

The index-sort pattern (sorting a table of parallel arrays)

FreeBASIC tables are often stored as several parallel arrays of equal length — one per column. Because vt_sort operates on a single array, you cannot sort all columns at once directly. The index-sort pattern solves this in three steps without duplicating the data. It also applies naturally to ZString arrays, which vt_sort cannot overload directly.

  1. Build a Long index array {0, 1, 2, …} — one entry per row.
  2. Sort the index array using vt_sort with a comparator that reads the real data via the index values.
  3. Either read in index order (non-destructive view) or call vt_sort_apply on every parallel array to physically rearrange them.
Because the comparator uses module-level Shared variables, FreeBASIC's lack of closures is not a problem — the comparator simply accesses the shared arrays directly. This technique works for any array type including ZString * N. 
' Table stored as parallel arrays:
Dim Shared g_item(3) As String
Dim Shared g_qty (3) As Long
g_item(0) = "eggs"   : g_qty(0) = 5
g_item(1) = "butter" : g_qty(1) = 2
g_item(2) = "milk"   : g_qty(2) = 10
g_item(3) = "bread"  : g_qty(3) = 3

' Index comparators peek into shared arrays via the index value:
Function cmp_by_qty(a As Long, b As Long) As Long
    Return g_qty(a) - g_qty(b)
End Function

' Step 1: build index array.
Dim idx(3) As Long
Dim i As Long
For i = 0 To 3 : idx(i) = i : Next i

' Step 2: sort the index, not the data.
vt_sort idx(), @cmp_by_qty   ' idx = {1,3,0,2} = butter,bread,eggs,milk

' Step 3a: read in index order (original arrays untouched):
For i = 0 To 3
    Print g_item(idx(i)); " "; g_qty(idx(i))
Next i

' Step 3b: physically rearrange both arrays using the same index:
vt_sort_apply g_item(), idx()
vt_sort_apply g_qty(),  idx()
' g_item() and g_qty() are now in qty-ascending order.

vt_file_copy

Copy a file or a full directory tree. The function auto-detects whether src is a file or a directory — no separate call needed.

File mode: the source file is read into memory and written to the destination in a single binary I/O operation. Fails if the destination already exists unless VT_FILE_OVERWRITE is passed.

Directory mode: the full directory tree is recreated under dst. If dst does not exist it is created; if it already exists the trees are merged. Individual files that already exist in dst are silently skipped unless VT_FILE_OVERWRITE is set. All entries are collected in a single Dir() pass before anything is written, so the global scan state is never corrupted by recursion.

Networking opt-in

TCP and UDP socket layer built on OS-native sockets: Winsock2 on Windows, POSIX on Linux. No external DLL is required — the Winsock2 import library (ws2_32) ships with FreeBASIC on Windows and POSIX sockets are always available on Linux. Enable with: 

#Define VT_USE_NET
#include once "vt/vt.bi"
  • vt_net_init / vt_net_shutdown — start / stop subsystem
  • vt_net_resolve / vt_net_ip_str — hostname lookup and IP formatting
  • vt_net_open / vt_net_close — create / destroy a socket
  • vt_net_nonblocking — toggle blocking mode
  • vt_net_connect — TCP client connect
  • vt_net_bind / vt_net_listen / vt_net_accept — TCP server setup
  • vt_net_send / vt_net_recv — TCP send and receive
  • vt_net_send_udp / vt_net_recv_udp — UDP send and receive
  • vt_net_ready — poll a socket (select wrapper)
  • vt_net_local_addr — query local endpoint

vt_net_init / vt_net_shutdown

Initialise and tear down the networking subsystem. vt_net_init must be called once before any other vt_net_* function. On Linux both calls are no-ops; on Windows vt_net_init calls WSAStartup and vt_net_shutdown calls WSACleanup.

Function vt_net_init() As Long Sub vt_net_shutdown()

Return value (vt_net_init)

0 on success.
-1 if Winsock initialisation failed (Windows only). 
If vt_net_init() <> 0 Then
    vt_print "Network init failed." & VT_NEWLINE
    End
End If
' ... program ...
vt_net_shutdown()

vt_net_resolve / vt_net_ip_str

Convert between host names and packed IP values. vt_net_resolve accepts both dotted-decimal strings ("192.168.0.1") and DNS host names ("example.com") and returns the IP as a packed Long in network byte order — the format expected by every other vt_net_* function. vt_net_ip_str converts a packed value back to a human-readable dotted-decimal string.

Function vt_net_resolve(hostname As ZString Ptr) As Long Function vt_net_ip_str(ip As Long) As String
ParameterTypeDescription
hostnameZString PtrDotted-decimal IP string or DNS host name to resolve.
ipLongPacked IP in network byte order, as returned by vt_net_resolve.

Return values

vt_net_resolve: packed IP on success, 0 on failure (unknown host or bad address).
vt_net_ip_str: dotted-decimal string (e.g. "93.184.216.34"), or empty string on failure. 
Dim ip As Long = vt_net_resolve("example.com")
If ip = 0 Then
    vt_print "Could not resolve host." & VT_NEWLINE
Else
    vt_print "Resolved: " & vt_net_ip_str(ip) & VT_NEWLINE
End If

vt_net_open / vt_net_close

Create and destroy a socket. Pass udp = 1 for a UDP datagram socket; the default creates a TCP stream socket. After use, always pass the handle to vt_net_close to release OS resources.

Dim sock As SOCKET = vt_net_open()

vt_net_nonblocking

Toggle non-blocking mode on a socket. In non-blocking mode vt_net_connect, vt_net_send, and vt_net_recv return immediately even when the operation cannot yet complete. Combine with vt_net_ready to integrate network I/O into a frame-rate game loop without stalling.

vt_net_connect

Connect a TCP socket to a remote host. The Nagle algorithm is disabled automatically (TCP_NODELAY) to minimise send latency, which is desirable for interactive game traffic. If the socket is in non-blocking mode (see vt_net_nonblocking), the call returns immediately with 0 while the connection is in progress; use vt_net_ready with for_write = 1 to detect when the connection has been established.

Function vt_net_connect(sock As SOCKET, ip As Long, port As Long) As Long
ParameterTypeDescription
sockSOCKETAn open TCP socket (from vt_net_open(0)).
ipLongRemote IP in network byte order, from vt_net_resolve.
portLongRemote port in host byte order (e.g. 80, 7000).

Return value

1 on success (blocking mode) or connection accepted immediately.
0 on failure, or when a non-blocking connection is still in progress. 
Dim sock As SOCKET = vt_net_open()
Dim ip   As Long   = vt_net_resolve("example.com")
If vt_net_connect(sock, ip, 80) = 0 Then
    vt_print "Connection failed." & VT_NEWLINE
    vt_net_close(sock)
    End
End If

vt_net_bind / vt_net_listen / vt_net_accept

Server-side TCP setup. Typical sequence: vt_net_bind assigns a local port, vt_net_listen begins queuing incoming connections, then vt_net_accept is called in a loop to obtain a new socket for each client. vt_net_bind is also used for UDP sockets that need a fixed local port.

Function vt_net_bind(sock As SOCKET, port As Long, ip As ULong = INADDR_ANY) As Long Function vt_net_listen(sock As SOCKET, backlog As Long = SOMAXCONN) As Long Function vt_net_accept(sock As SOCKET, remote_ip As Long Ptr = 0, remote_port As Long Ptr = 0) As SOCKET
ParameterTypeDescription
sockSOCKETThe server / listening socket.
portLongLocal port to bind to, in host byte order.
ipULongLocal interface IP in network byte order. Default INADDR_ANY binds to all interfaces.
backlogLongMaximum pending connections queued before the OS starts refusing. Default SOMAXCONN (OS maximum).
remote_ipLong PtrIf non-null, receives the client's IP (network byte order) when a connection is accepted.
remote_portLong PtrIf non-null, receives the client's port (host byte order) when a connection is accepted.

Return values

vt_net_bind, vt_net_listen: 1 on success, 0 on failure.
vt_net_accept: a new SOCKET handle for the accepted connection, or INVALID_SOCKET if no connection is pending (non-blocking socket) or on error.
TCP_NODELAY is set automatically on every socket returned by vt_net_accept, matching the behaviour of vt_net_connect. Manage each accepted socket independently and close it with vt_net_close when the client disconnects. 
Dim srv  As SOCKET = vt_net_open()
vt_net_bind   srv, 7000
vt_net_listen srv

vt_print "Waiting for connection..." & VT_NEWLINE
vt_present()

Dim client As SOCKET
Dim cip    As Long
Dim cport  As Long
Do
    client = vt_net_accept(srv, @cip, @cport)

vt_print "Client: " & vt_net_ip_str(cip) & ":" & cport & VT_NEWLINE

vt_net_send / vt_net_recv

Send and receive data on a connected TCP socket. Both functions map directly to the OS send and recv calls. In blocking mode vt_net_recv waits until data arrives or the connection closes; call vt_net_ready first to avoid stalling the game loop.

Function vt_net_send(sock As SOCKET, buf As ZString Ptr, nbytes As Long) As Long Function vt_net_recv(sock As SOCKET, buf As ZString Ptr, nbytes As Long) As Long
ParameterTypeDescription
sockSOCKETConnected TCP socket.
bufZString PtrPointer to the data buffer to send from or receive into.
nbytesLongNumber of bytes to send, or the maximum bytes the receive buffer can hold.

Return values

vt_net_send: number of bytes actually sent, or ≤ 0 on error or disconnection.
vt_net_recv: number of bytes received. 0 means the peer closed the connection gracefully; < 0 is an error.
A single vt_net_send call may send fewer bytes than requested — a robust send loop should retry until the full byte count has been delivered. vt_net_recv may similarly return fewer bytes than nbytes even when more data is on the way; reassemble as needed.

Dim n      As Long

n = vt_net_recv(sock, @rxbuf, 256)
If n > 0 Then
    rxbuf[n] = 0
    vt_print "Received: " & rxbuf & VT_NEWLINE
ElseIf n = 0 Then
    vt_print "Server closed connection." & VT_NEWLINE
End If

Send and receive UDP datagrams. A UDP socket does not need to be connected; each send call specifies the destination explicitly. To receive, the socket should first be bound to a local port with vt_net_bind.

vt_net_ready

Non-blocking readiness poll on a single socket using select(). Use this before vt_net_recv or vt_net_accept in a game loop to avoid blocking the frame. Pass timeout_ms = 0 for an instant check, or timeout_ms = -1 to block until the socket is ready.

Function vt_net_ready(sock As SOCKET, for_write As Byte = 0, timeout_ms As Long = 0) As Long
ParameterTypeDescription
sockSOCKETSocket to poll.
for_writeByte0 = check whether data is available to read (default). 1 = check whether the send buffer has space.
timeout_msLongTimeout in milliseconds. 0 = return immediately. -1 = block until the socket is ready.

Return value

1 if the socket is ready for the requested operation.
0 if the timeout expired with no activity.
-1 on error. 
' Non-blocking receive in a game loop:
Dim rxbuf     As ZString * 257
Dim connected As Long = 1
Dim k         As ULong
Dim n         As Long
Do
    k = vt_inkey()

    If vt_net_ready(sock) Then
        n = vt_net_recv(sock, @rxbuf, 256)
        If n > 0 Then
            rxbuf[n] = 0
            ' process rxbuf
        ElseIf n = 0 Then
            connected = 0  ' server closed the connection
        End If
    End If

    ' ... draw frame, handle input ...
    vt_sleep 16
Loop While connected
